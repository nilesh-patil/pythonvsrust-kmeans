# Feature 1 — k-means++ initialization

## Why
The custom Python and Rust impls currently use **random** centroid initialization, while sklearn uses **k-means++** with `n_init=10`. This confounds the comparison: sklearn's better silhouette (0.935 vs 0.62 for both custom impls) is partly explained by the smarter init. Adding k-means++ to the custom impls makes the cross-language comparison apples-to-apples.

## Algorithm (Arthur & Vassilvitskii 2007)
1. Pick the first centroid uniformly at random from `X`.
2. For each subsequent centroid: compute D(x)² (squared distance from each point to the nearest already-chosen centroid). Sample the next centroid from the points with probability proportional to D(x)².
3. Repeat until `k` centroids are chosen.

## Python contract
- Add `init: str = "random"` parameter to `KMeansClustering.__init__` (accepts `"random"` or `"k-means++"`).
- Implement `_initialize_centroids_kpp(X)` producing `(k, n_features)` centroids.
- Existing random path remains unchanged (preserves baseline).
- CLI: new `--init {random,k-means++}` flag, default `"random"` to avoid breaking existing benchmarks.

## Rust contract
- Mirror the API: `enum InitMethod { Random, KMeansPlusPlus }`; `KMeans::new` takes it.
- CLI: `--init random|k-means++`, default `random`.
- Use squared Euclidean for the D² sampling — no `.sqrt()` is needed and it avoids drift.

## Tests (write first)

### Python (`tests/test_kmeans_init.py`)
- `test_kpp_returns_k_distinct_points`: for `X` with 100 points, request `k=5`, assert returned shape == `(5, n_features)` and all rows belong to `X`.
- `test_kpp_seed_reproducibility`: same seed → identical centroids.
- `test_kpp_beats_random_inertia_on_separated_blobs`: generate 4 well-separated blobs in 2D; over 20 runs, mean inertia of k-means++ ≤ mean inertia of random init.
- `test_kpp_handles_k_equals_n`: when `k == n_samples`, returns each point exactly once.

### Rust (`src/rust_impl/src/kmeans.rs` `#[cfg(test)]`)
- `test_kpp_picks_k_distinct_indices`
- `test_kpp_deterministic_with_seed`
- `test_kpp_lower_inertia_than_random_on_blobs`

## Acceptance criteria
- All new tests pass (`pixi run pytest tests/test_kmeans_init.py`, `cargo test`).
- Existing benchmarks still pass with `--init random` (default unchanged).
- `results/init_comparison.png` is committed and shows k-means++ ≤ random inertia on every dataset.
- Both READMEs (Python and Rust impl) document the new flag.

## Out of scope
- Multiple restarts (`n_init`) — that's a separate concern.
- Greedy k-means++ (sklearn's `init=auto`) — vanilla weighted sampling only.
